# コードレビューチェックリスト

## アーキテクチャ

- [ ] **ADR 準拠**: 既存の Architecture Decision Records に従っているか
- [ ] **レイヤー分離**: Domain/Data/UI の依存関係が正しいか
- [ ] **MVI パターン**: Intent → State の一方向データフローか

## Compose UI

- [ ] **4層構造**: Container → Screen → Content → Component の分離
- [ ] **Preview 作成**: Screen/Content/Component に Preview があるか
- [ ] **AppTheme 使用**: MaterialTheme 直接使用していないか
- [ ] **colorScheme 経由**: ハードコードカラーを使用していないか
- [ ] **contentDescription**: インタラクティブ要素に設定されているか
- [ ] **remember 使用**: 高コスト計算がキャッシュされているか
- [ ] **Clock 禁止**: Screen 以下で Clock.System を使用していないか

## コード品質

- [ ] **命名規則**: クラス名、関数名、変数名が適切か
- [ ] **コメント**: 複雑なロジックに説明があるか
- [ ] **エラーハンドリング**: 例外処理が適切か
- [ ] **Null 安全**: 適切な Null チェックがあるか

## テスト

- [ ] **テスト追加**: 新機能にテストがあるか
- [ ] **既存テスト通過**: CI が通っているか
- [ ] **DisplayName**: テストに日本語説明があるか

## ドキュメント

- [ ] **REQUIREMENTS 準拠**: 仕様書の要件を満たしているか
- [ ] **ドキュメント更新**: 必要な場合は更新されているか

## セキュリティ

- [ ] **秘密情報**: API キー等がハードコードされていないか
- [ ] **入力検証**: ユーザー入力が適切に検証されているか

## パフォーマンス

- [ ] **不要な再レンダリング**: Compose の再コンポジションが最適化されているか
- [ ] **メモリリーク**: 適切なライフサイクル管理がされているか

---

## 使い方

1. PR のコメント欄にこのチェックリストをコピー
2. 該当する項目にチェックを入れる
3. 問題がある項目にコメントを追加
