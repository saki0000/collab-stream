# ADR 004: 手動同期方式採用

## Status
Accepted

## Context

CollabStreamにおける動画同期の実装方式について検討が必要：

**要件**:
- 2つのストリームアーカイブを同期再生
- 秒レベルの同期精度で十分
- シンプルな操作性

**技術的制約**:
- URL パラメータ変更による同期実装
- サーバーサイド同期は不要
- リアルタイム通信は不要

同期方式として以下を検討：

1. **自動同期**: 常時同期状態を監視し自動補正
2. **手動同期**: ユーザーがボタン押下で同期実行
3. **ハイブリッド**: 手動開始 + 自動維持
4. **イベント駆動**: 特定イベント発生時に同期

## Decision

**手動同期方式** を採用する。

### 実装方式
- 同期ボタンをUI に配置
- ボタン押下時にURL パラメータを調整
- タイムスタンプを両方の動画に適用

### 採用理由

1. **要件適合**: ボタン押下での同期で十分
2. **実装簡素**: URL パラメータ変更のみで実現
3. **リソース効率**: 常時監視が不要
4. **ユーザー制御**: 必要時のみ同期実行
5. **デバッグ容易**: 明確な同期タイミング

## Consequences

### Positive
- **シンプル実装**: 複雑な同期ロジック不要
- **低CPU使用**: 常時監視処理が不要
- **明確操作**: ユーザーが同期タイミングを制御
- **デバッグ容易**: 同期実行が明確
- **バッテリー効率**: バックグラウンド処理最小

### Negative
- **手動操作**: ユーザーが同期ボタンを押す必要
- **精度制限**: リアルタイム補正なし
- **一時的ずれ**: 時間経過での自然なドリフト発生

### Neutral
- **将来拡張**: 必要に応じて自動同期に拡張可能
- **UX影響**: 手動操作の受容性はユーザーテストで評価
